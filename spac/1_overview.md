# 1. 개요

관리 대상 데이터를 **도메인(Domain)** 단위로 최상위 분류하고, 그 아래로 **트리 구조의 분류체계(Classification Tree)** 를 구성한다. 트리의 각 노드는 자신만의 **필드(Field)** 를 정의할 수 있으며, 정의된 필드는 하위 노드에 자동으로 **상속**된다. 필드는 관리자가 자유롭게 추가/수정할 수 있는 동적 스키마여야 한다.

예시:

임직원 (Domain)
├─ 정규직 (Node)
│ └─ 수습 (Node)
└─ 비정규직 (Node)
  ├─ 계약직 (Node)
  └─ 파견직 (Node)

`임직원`에 "입사일" 필드를 추가하면, `정규직`, `수습`, `비정규직`, `계약직`, `파견직` 전부 해당 필드를 갖게 된다.

---

# 2. 핵심 개념 정의

| 개념 | 설명 |
|---|---|
| **Domain** | 분류체계의 루트. 데이터의 최상위 분류 단위 (예: 임직원, 고객, 상품). 이 도메인에 속한 레코드를 식별/표시하기 위한 기준 필드(ID, 이름, 설명)를 보유한다. |
| **ClassificationNode** | 트리를 구성하는 노드. 도메인 자체도 트리의 root node로 취급된다. |
| **FieldDefinition** | 특정 노드에 정의된 필드 (이름, 타입, 옵션 등). |
| **EffectiveField** | 특정 노드 기준으로, 자기 자신 + 모든 상위 노드로부터 상속받은 필드를 합친 최종 필드 집합. |
